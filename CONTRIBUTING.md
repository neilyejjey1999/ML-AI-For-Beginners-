# Contributing to ML-AI-For-Highschool 🎉

We'd love your contributions! This resource is built by the community, for the community.

## Ways to Contribute

### 📝 Found a typo or error?
1. Open an [Issue](https://github.com/neilyejjey1999/ML-AI-For-Highschool/issues) describing the error
2. (Optional) Submit a PR with the fix

### 💡 Have a suggestion?
1. Open an Issue with your idea
2. Let's discuss and improve it together!

### 📚 Want to add a new notebook?
1. Check existing notebooks first (avoid duplicates)
2. Follow the style and format of existing notebooks
3. Include explanations, code examples, and exercises
4. Test it thoroughly before submitting

### 🐛 Found a bug in the code?
1. Open an Issue with details
2. Include: What went wrong, expected vs actual behavior
3. Submit a PR with the fix if you can!

## How to Submit a Pull Request (PR)

### For GitHub Beginners:

**Step 1: Fork the repository**
- Click "Fork" button in top right
- This creates your own copy

**Step 2: Clone your fork**
```bash
git clone https://github.com/YOUR-USERNAME/ML-AI-For-Highschool.git
cd ML-AI-For-Highschool
```

**Step 3: Create a new branch**
```bash
git checkout -b my-improvement
```

**Step 4: Make your changes**
- Edit files as needed
- Test everything works

**Step 5: Commit and push**
```bash
git add .
git commit -m "Add/fix [what you changed]"
git push origin my-improvement
```

**Step 6: Open a Pull Request**
- Go to original repo on GitHub
- Click "New Pull Request"
- Select your branch
- Add description of changes
- Submit!

## Style Guide 📋

Please follow these conventions:

### Jupyter Notebooks
- Start with a clear title: `# Topic Name 📚`
- Include learning objectives
- Add explanations before code
- Include working code examples
- End with key takeaways
- Add practice exercises
- Link to next notebook

### Code
- Use clear variable names: `student_score` not `ss`
- Add comments for complex logic
- Follow PEP 8 Python style
- Test code before submitting

### Documentation
- Use clear, beginner-friendly language
- Avoid jargon (or explain it)
- Use examples from everyday life
- Keep it engaging!

## Content Guidelines 📖

✅ **Do:**
- Explain concepts simply
- Use visualizations
- Include real-world examples
- Make it fun and engaging
- Test all code thoroughly
- Provide practice problems

❌ **Don't:**
- Use overly complex math
- Assume prior knowledge
- Skip explanations
- Make it boring
- Include untested code
- Copy content without attribution

## Notebook Template 📝

Start new notebooks with this structure:

```markdown
# Topic Name 🎓

Welcome! In this notebook, we'll learn [what they'll learn].

## What You'll Learn
- Concept 1
- Concept 2
- Concept 3

## Explanation Section

[Clear explanation with examples]

## Code Examples

[Working Python code with comments]

## Key Takeaways 🎯

✅ Point 1
✅ Point 2
✅ Point 3

## Practice Exercise 💪

Try this challenge...

## Next Steps 🚀

Next: [Link to next notebook]
```
